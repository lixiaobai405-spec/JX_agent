# 错误码参考

## 错误响应格式

所有错误响应遵循统一格式：

```json
{
  "detail": "错误描述信息",
  "error_code": "ERROR_CODE",
  "timestamp": "2026-03-18T10:00:00Z"
}
```

## 通用HTTP错误

| HTTP状态码 | 说明 | 示例场景 |
|-----------|------|----------|
| 400 | 请求参数错误 | 缺少必填字段、数据格式错误 |
| 401 | 未认证 | 缺少 token、token 无效 |
| 403 | 无权限 | 角色权限不足 |
| 404 | 资源不存在 | 请求的资源 ID 不存在 |
| 409 | 资源冲突 | 唯一性约束冲突 |
| 422 | 数据验证失败 | Pydantic 验证失败 |
| 429 | 请求过于频繁 | 超出限流限制 |
| 500 | 服务器内部错误 | 未预期的系统错误 |
| 503 | 服务不可用 | 数据库连接失败、外部服务不可用 |

## 错误处理最佳实践

### 客户端处理

1. **检查 HTTP 状态码**：首先判断请求是否成功
2. **解析 error_code**：根据错误码进行特定处理
3. **显示 detail**：向用户展示友好的错误信息
4. **重试策略**：对于 5xx 错误可以考虑重试

### 示例代码

```javascript
try {
  const response = await fetch('/api/v1/auth/login', {
    method: 'POST',
    body: JSON.stringify({ username, password })
  });

  if (!response.ok) {
    const error = await response.json();

    switch (error.error_code) {
      case 'AUTH_001':
        alert('用户名或密码错误');
        break;
      case 'AUTH_005':
        alert('登录尝试过于频繁，请稍后再试');
        break;
      default:
        alert(error.detail);
    }
    return;
  }

  const data = await response.json();
  // 处理成功响应
} catch (err) {
  console.error('网络错误', err);
}
```
